# Humanoid Robots Book Constitution

## 1. Project Theme & Purpose

### Execution Restrictions
Claude Code must:
- Only execute tasks related to textbook content creation.
- Must NOT:
  - Start chatbot development
  - Start hardware installation steps
  - Create automation for ROS, robots, or sensors
Unless explicitly defined in /specs/features/.

### Deployment Standard
All textbook deployments MUST use **Vercel** as the hosting platform.  
Claude Code must generate deployment files/configurations only for Vercel.

### 1.1 Theme
**Digital Brain → Physical Body (Embodied Intelligence)**

### 1.2 Purpose
Build a full **Docusaurus textbook** on Physical AI & Humanoid Robotics using:
- Claude Code
- SpecKit Plus
- Docusaurus

Automation must create missing files/folders automatically.

### 1.3 Deliverables
- Public GitHub repo
- Live Docusaurus book

## 2. Communication & Language Standards

### 2.1 Languages
- Code → English
- Docs → English
- Technical terms → English
- File/folder names → English (kebab-case)
- User-facing responses → Roman Urdu

### 2.2 Commenting
- ALL code comments must be English
- Terminology must stay consistent

---

## 3. Project Organization Standards

### 3.1 Directory Rules
- Organize by **feature/module**, not filetype
- Max 3–4 nesting levels
- Root must stay clean (configs only)
- Related files stay together

### 3.2 Folder Hygiene
- Delete unused files
- Use `.gitignore`
- Keep separate:
  - source code
  - documentation
  - configuration

### 3.3 Documentation Placement
- Specs → root or `/specs`
- Textbook content → `/docs`
- README always updated
